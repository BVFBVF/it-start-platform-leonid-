# it-start-platform-leonid-

Стек: Next.js 14 (App Router), TypeScript, Tailwind CSS

Бэкенда нет — всё на моковых данных.
Структуру данных придумай сам, 
используй файл src/lib/mock-data.ts

## Фича 1: Глобальный поиск /search

- Строка поиска с результатами в реальном времени
- Поиск по сотрудникам (имя, стек), задачам (название), 
  менторам (имя)
- Результаты сгруппированы по категориям: 
  Сотрудники / Задачи / Менторы
- Каждый результат кликабельный → ведёт на нужную страницу
- Если ничего не найдено — пустое состояние с иконкой
- Поиск через useDebounce — запрос не на каждый символ,
  а через 300мс после остановки печати

## Фича 2: Лента активности /activity

- Хронологический список событий на платформе
- Типы событий:
  - Сотрудник добавлен
  - Статус изменён (active → blocked и т.д.)
  - Ревью создано
  - Задача завершена
  - PR смержен
  - Сотрудник готов к аутстаффу
- У каждого события: иконка типа, текст, 
  имя участника, дата и время
- Фильтрация по типу события
- Группировка по дням: "Сегодня", "Вчера", "12 июня"
- Бесконечная подгрузка (кнопка "Загрузить ещё")

## Фича 3: Страница команды /team

- Общая структура команды в виде дерева:
  Ментор → его джуны
- Карточка ментора: имя, фото, специализация, 
  нагрузка (сколько джунов)
- Под каждым ментором — карточки его джунов:
  имя, статус, readiness score
- Переключатель вида: дерево / таблица
- В табличном виде: все сотрудники одним списком
  с колонками (имя, ментор, статус, score, задача)
- Сортировка по колонкам в табличном виде

## Требования

- Компоненты декомпозированы
- Общие компоненты в src/components/ui/
- Типизация везде, any не использовать
- Адаптивная вёрстка (мобилка + десктоп)
- Каждая фича — отдельная ветка и PR в main
